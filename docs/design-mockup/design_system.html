01 — Logo

Brand

A chunky Z mounted on a hard offset plate. The red square is the "issue dot" — it always sits on the bottom-right and is the only place red appears in the brand.

Construction

Three layers, no curves.

  • FACE  —  yellow square, 3px ink border
  • PLATE —  ink square offset +4 / +4
  • DOT   —  red 7px, bottom-right, 2px border
Don't

Hard rules.

  • No gradients, no glow, no rounded corners
  • Never re-color the Z — only the FACE swaps
  • Never remove the offset plate
  • Minimum size: 24×24 px

07 — REST & MCP

Surface

Every REST endpoint has a matching MCP tool. Agents can either call POST /issues directly, or invoke the tool zdzira.issue.create — same args, same response, same SQLite file.

REST endpoints

GET
/issues
List · filter by status, priority, q=
GET
/issues/:id
Single issue with links & activity
POST
/issues
Create · title required, body optional
PATCH
/issues/:id
Partial update · status, priority, title…
DEL
/issues/:id
Soft-delete by default, ?hard=1 to purge
POST
/issues/:id/comments
Append a comment · supports markdown

MCP tool call

// .mcp.json
{
  "mcpServers": {
    "zdzira": {
      "command": "zdzira",
      "args": ["mcp"],
      "env": { "ZDZIRA_DB": "./.zdzira/db.sqlite" }
    }
  }
}

// agent invocation
tool: zdzira.issue.create
args: {
  "title": "Fix sqlite write contention",
  "priority": "P0",
  "body":  "Parallel agents block >3s"
}

// response
{
  "id":       "ZDR-0142",
  "status":   "open",
  "priority": "P0",
  "created":  "2026-05-26T14:02:11Z"
}
